  <behavior>
    Valid transitions (machine.apply returns target state):
      - TODO → IN_PROGRESS returns TaskStatus.IN_PROGRESS
      - IN_PROGRESS → PEER_REVIEW returns TaskStatus.PEER_REVIEW
      - PEER_REVIEW → REVIEW returns TaskStatus.REVIEW
      - PEER_REVIEW → IN_PROGRESS returns TaskStatus.IN_PROGRESS (rejection path)
      - REVIEW → APPROVED returns TaskStatus.APPROVED

    Invalid transitions (machine.apply raises InvalidTransitionError):
      - TODO → APPROVED raises
      - TODO → PEER_REVIEW raises
      - APPROVED → TODO raises
      - IN_PROGRESS → APPROVED raises
      - REVIEW → IN_PROGRESS raises

    Error message:
      - InvalidTransitionError str contains "todo" when from=TODO
      - InvalidTransitionError str contains "approved" when to=APPROVED
      - Both state values appear in the same message

    Terminal state:
      - APPROVED has no valid outgoing transitions
      - machine.apply(APPROVED, any) always raises

    Rejection path semantics:
      - PEER_REVIEW → IN_PROGRESS is VALID (this is how rejection works)
      - "rejected" never appears as a TaskStatus value or transition target
  </behavior>
