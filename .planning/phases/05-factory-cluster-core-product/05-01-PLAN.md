Dockerfile pattern (per CONTEXT.md Dockerfile/Image Strategy):
```dockerfile
# Default (no glibc-dependent roles)
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY runtime/ runtime/
COPY config/ config/
CMD ["python", "-m", "runtime.cli", "run-agent"]

# Alternate (any role has requires_glibc=True)
FROM ubuntu:22.04
RUN apt-get update && apt-get install -y python3.12 python3-pip && rm -rf /var/lib/apt/lists/*
WORKDIR /app
COPY requirements.txt .
RUN pip3 install --no-cache-dir -r requirements.txt
COPY runtime/ runtime/
COPY config/ config/
CMD ["python3", "-m", "runtime.cli", "run-agent"]
```

requirements.txt pattern: one package name per line, collected from all roles' tool_allowlist. Deduplicated and sorted. Always includes the runtime's own dependencies as a baseline (anthropic, pydantic, aiosqlite, click, tabulate).

Stagger offset formula: boss=0, then offset_i = i * (interval_seconds / total_roles) for each worker role.
Total roles = len(roles) + 2 (boss + critic structural roles always added).

Key implementation notes:
- Use `yaml.dump({...}, default_flow_style=False, allow_unicode=True)` for all YAML output — never f-string YAML
- Stagger offset per role: `offset_i = round(i * (interval_seconds / total_count), 1)` where total_count = len(roles) + 2 (boss + critic) and i is 0-indexed for all services (boss=0, critic=1, role_0=2, ...)
- render_agent_yaml accepts `cluster_config: dict | None = None` for backwards compatibility (ignored for stagger — use stagger_offset_seconds param). Full signature: `render_agent_yaml(role: RoleSpec, cluster_config: dict | None = None, stagger_offset_seconds: float = 0.0) -> str`
- docker-compose.yml: each service has build/command/volumes/env_file. Named volume "cluster-data" at bottom. Boss and critic are always included. Each service's command is `python -m runtime.cli run-agent --role <role_name>`.
- render_launch_sh: uses textwrap.dedent for clean multiline shell script. Must contain the ANTHROPIC_API_KEY check, `docker compose build`, `docker compose up -d`
- render_dockerfile: inspect `any(r.requires_glibc for r in roles)`. If True, use ubuntu:22.04 base with python3.12 apt install. If False, use python:3.12-slim. Both variants COPY requirements.txt and RUN pip install, then COPY runtime/ and config/.
- render_requirements_txt: collect all tool_allowlist entries across all roles, deduplicate, sort. Prepend baseline packages: `anthropic`, `pydantic`, `aiosqlite`, `click`, `tabulate`. Return newline-joined string.
- copy_runtime: `shutil.copytree(Path(__file__).parent.parent / "runtime", dest_dir / "runtime", dirs_exist_ok=True)`
- Atomic writes: use Path.replace() pattern for any direct file writes (copy_runtime uses shutil which handles its own atomics)
